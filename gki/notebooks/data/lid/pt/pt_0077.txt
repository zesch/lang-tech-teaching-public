A Wikipédia atualmente roda em clusters de servidores dedicados Linux (principalmente Ubuntu),[210][211] com algumas máquinas OpenSolaris com sistemas de arquivos ZFS. Em dezembro de 2009, havia 300 na Flórida, Estados Unidos, e 44 em Amsterdã, nos Países Baixos.[212] A Wikipédia empregava um único servidor até 2004, quando a configuração do servidor foi ampliada para uma arquitetura em multicamadas distribuídas. Em janeiro de 2005, o projeto funcionava através de 39 servidores dedicados localizados na Flórida. Esta configuração inclui um servidor de banco de dados único mestre executando o MySQL, múltiplos servidores de banco de dados escravos, 21 servidores web rodando o Apache HTTP Server e sete servidores de cache rodando o software Squid.
